중앙 지식 저장소 경로(`WIKI_REPO_PATH`)는 이 파일에 적지 않는다. 실행할 때마다 개인 설정 파일에서 읽는다
(설치 방법은 저장소 루트 README.md의 "설치" 절 참고).

`log/*.md`의 frontmatter(`start_time`, `end_time`, `project`, `branch`)와 본문(특히 `## Jira 업데이트` /
`## Confluence 업데이트` 절)을 바탕으로, 지정한 기간의 업무 보고 초안과 프로젝트별 통계를 만든다.
이 커맨드는 읽기 전용이다 — 어떤 파일도 새로 만들거나 수정하지 않고, 결과를 대화창에만 출력한다.

다음 순서로 실행한다:

1. `cat ~/.config/llm_wiki/repo_path`를 실행해서 `WIKI_REPO_PATH`를 얻는다.
   파일이 없거나 비어있으면, 다음과 같이 안내하고 **중단한다**:
   "먼저 `mkdir -p ~/.config/llm_wiki && echo '<본인의 llm_wiki clone 절대경로>' > ~/.config/llm_wiki/repo_path`로 한 번 설정해주세요."
2. `date +'%Y-%m-%d'`를 실행해서 오늘 날짜를 구한다.
3. `$ARGUMENTS`로 기간 힌트를 해석한다:
   - 비어있으면 기본값: 최근 7일 (오늘 포함, 오늘-6일 ~ 오늘).
   - "이번 주", "지난 N일", "YYYY-MM-DD ~ YYYY-MM-DD" 같은 자연어/날짜 범위를 받아 실제 시작일·종료일로 변환한다.
     애매하면 추측하지 말고 사용자에게 되묻는다.
4. `<WIKI_REPO_PATH>/log/*.md`를 모두 열어 frontmatter의 `start_time`(또는 `end_time`)이 3번의 기간 안에
   드는 파일만 골라낸다 (파일명의 날짜 prefix로 1차로 후보를 좁히고, frontmatter로 정확히 재확인한다).
5. 대상 로그가 없으면 "해당 기간에 기록된 log가 없습니다"라고 보고하고 종료한다.
6. 대상 로그들을 각각 읽어 `project`별로 그룹핑한다. 각 로그에서:
   - 본문 요약(무엇을 했는지, 어떤 결정을 했는지 — 잡담이나 세션 진행상황이 아니라 결과 위주로 1~3줄)
   - `## Jira 업데이트` 절이 있으면 언급된 이슈 키와 상태를 함께 뽑는다.
7. 아래 형태로 **업무 보고 초안**을 만들어 출력한다 (파일로 저장하지 않는다):
   ```
   # <시작일> ~ <종료일> 업무 요약

   ## <project A>
   - (요약 bullet들, 관련 Jira 이슈 키 포함)

   ## <project B>
   - ...
   ```
8. 이어서 **프로젝트별 시간 통계** 표를 만든다: 각 project의 세션 수, 그리고 각 세션의
   `end_time - start_time` 차이를 합산한 대략의 소요 시간(시간 단위, 소수점 1자리)을 표로 보여준다.
   `start_time == end_time`인 세션(순수 대화/조사)은 소요 시간 계산에서 제외하거나 별도로 표시한다.
9. 마지막에 "이 요약은 log/에 기록된 스냅샷 기반이므로, 최종 보고 전에는 직접 검토·수정할 것"이라고
   한 줄 덧붙인다.
